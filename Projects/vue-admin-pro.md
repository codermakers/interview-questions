### 二、开发问题

**防止重复提交**：

- 前端控制
  - 防抖/节流
  - axios 设置请求间隔时间 1000,重复请求提示"数据正在提交中，切勿重复提交，请稍后!"(针对 post 和 put 请求，具体可参考若依)
  - 按钮设置 1 秒后置灰 disable 或者不可见(可通过自定义指令实现);
  - lock/flag 标志控制
  - loading(按钮加 loading)

- 后端控制
- - 缓存
  - 索引
  - token
  - redis 计时器等

**文件导入、导出、预览**

- 文件导入
- 文件导出(以导出 Excel 为例)
  - 纯前端生成 Excel
  - 后端生成 Excel

    - 返回文件流

- 文件预览
  - PDF 预览
    - 思路
      - iframe、
      - 第三方 PDF 预览库；vue-pdf

  - word、Excel、PPT 预览

**登录鉴权**

- 登录
  - 主动登出(时间戳)
  - 被动登出(后端返 401 HTTP 状态码)
  - 双 Token 机制(用户无感登录)
  - 第三方登录(QQ、微信、gitee...)
  - SSO 单点登录
- 注销
  - 调用登出接口 -> 清除本地存储 token,Vuex 中的 token -> 返回登录页

流程： 用户 -> 触发登录 -> 获取 accessToken ->存入本地 sessionStorage ->axios 请求头 header 增加 Authorization 字段： Bearer + getToken()

**权限管理**

- 权限管理的 2 种模型：
  - RBAC(基于角色的权限控制模型)
  - ABAC(基于属性的权限控制模型)

- 权限控制
  > 前端权限控制：仅能控制视图 UI 层面的可见与不可见或者禁用；并不能防止接口被非法调用; 保险起见，后端需要 对 API 接口做权限控制(权限注解)
  - 页面/菜单/路由权限
  - 按钮权限
    - 按钮置灰，并且给出用户相应的提示`用户可能通过绕过，比如修改按钮可用`
    - 按钮可见与不可见(推荐)
  - 数据权限(后台控制)
    - 用户数据过滤控制
