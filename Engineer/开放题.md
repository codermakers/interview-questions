## 二、问题解决和软技能类 (考察你能不能融入团队)

### 2.1 线上突然出现了一个Bug,你怎么处理

- 快速止损: 如果bug是致命的，先回滚，否则下一个版本修复
- 定位问题: 通过异常监控、日志，复现Bug，定位问题
- 紧急修复: 紧急修复bug，测试仿真环境通过，上线
- 事后复盘: 故障复盘，制定规避方案，沉淀到技术文档，避免同类问题出现

### 2.2 产品提了一个不合理的需求，你会怎么处理？

- 先找产品明确需求背景和目标，确认是否理解错需求
- 把技术实现的成本、工期、影响告知产品或者给出替代方案
- 如果还是有分歧，拉齐相关负责人，在保证业务需求的前提下，选择最优方案？

### 2.3 项目需求延期怎么办？

- 明确延期的原因？ 技术实现难度高(替代方案)、资源不足(加人或加时间)、需求变动(调时间)？

### 2.4 你怎么看待加班 ？

```HTTP
 首先，我不排斥必要的加班，比如项目上线，紧急bug，业务高峰期，肯定会加班；
 但我更希望通过提升开发效率、做好项目规划、优化技术方案，减少无意义的无效加班，平衡好工作和生活，长期来看也能保证更高的工作效率。
```

### 2.5 团队来了新人，如何带他上手？

```HTTP
1. 先给新人做体系化的介绍：团队业务、技术栈、开发规范、项目结构、协作流程，让他有整体的认知；
2. 给他安排循序渐进的需求，先从简单的页面、组件开发入手，再逐步接触核心业务，帮他建立信心；
3. 定期 1v1 沟通，解答他的疑问，CodeReview 里重点帮他指出问题，给出优化建议；
4. 鼓励他主动提问、主动思考，把自己踩过的坑、沉淀的文档分享给他，帮他少走弯路，快速融入团队。

```

### 2.6 你是如何学习新技术的

```HTTP
 1、日常关注前端社区掘金、MDN等，了解行业动态
 2、学习新技术，先看官方文档，写Demo验证，在项目中实践
 3、把学习到内容沉淀到技术文档，分享到团队

```
